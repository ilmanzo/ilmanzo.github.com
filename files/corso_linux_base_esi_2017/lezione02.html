mount, umount

esercizio:

  • creiamo un file da 100MB con il comando 'dd'
  • trattiamo il file come se fosse un disco e lo 'formattiamo' con un filesystem ext4
  • creiamo un mount point in /mnt/nuovodisco
  • montiamo il nuovo filesystem e dentro scriviamo un file prova.txt
  • smontiamo il filesystem

per chi vuole: preparare due "dischi" e fare un device RAID1 (mirroring)

informazioni su

http://www.bufferoverflow.it/2009/12/30/usare-i-loop-device-per-creare-filesystem-in-raid-su-linux/

https://raid.wiki.kernel.org/index.php/RAID_setup#RAID-1

quale filesystem scegliere ?

esistono diversi tipi di filesystem, ognuno ha pregi e difetti, e risulta piu' adatto a cose specifiche; qualche esempio:

  • VFAT, NTFS (mondo Windows)
  • HFS (sui sistemi Apple)
  • ext3,ext4 : il fs tradizionale linux
  • xfs, btrfs : il nuovo che avanza
  • zfs: per sistemi enterprise (Oracle)
  • unionfs, jffs2: per esigenze specifiche
  • nfs, smbfs: accesso a dati in rete

eccetera https://en.wikipedia.org/wiki/Comparison_of_file_systems
