utenti e gruppi

in unix ogni utente e' identificato da un UID

uid = 0 per default e' ROOT

gli uid "bassi" fino a N (dipende dalla distribuzione) sono riservati per gli utenti di sistema

insieme di utenti = gruppo

ogni gruppo ha un GID

ogni utente ha un gruppo predefinito ma puo' far parte anche di altri gruppi

ogni utente ha delle caratteristiche: UID, GID, home directory, shell predefinita

nice, limits, quota

l'amministratore di sistema puo' e deve impostare dei limiti alle risorse che ogni utente puo' usare

in termini di cpu, disco, memoria, processi, connessione di rete eccetera

comandi: quota, ulimit, nice, cpulimit ... concetti: cgroups, traffic shaping

permessi sui file

  • R = lettura (Read)
  • W = scrittura (Write)
  • X = esecuzione (eXecute)
  • UGO: utente proprietario (User), gruppo (Group), altri (Other)
  • man chmod

permessi sulle directory

  • R significa elencare il contenuto della directory
  • W significa creare, rinominare, cancellare file nella directory
  • X significa poter entrare nella directory e accedere ai file contenuti in essa
