un semplice servizio di rete

esercizio: scriviamo un web server!

  • httpd e' un servizio di rete che riceve una richiesta sulla porta 80 (standard http) e

  • restituisce al client una "pagina" html presa dal filesytem, esempio /home/testuser/web/index.html

  • opzionale: nella pagina ci possono essere contenuti variabili (ad esempio la data di sistema e il peso letto da una bilancia collegata al raspberry)

extra points...

  • fare in modo che nella pagina ci sia scritto quante volte l'abbiamo vista

soluzione esercizio: webserver

il vero e proprio script che legge la richiesta e confeziona la risposta

$ cat webserver.sh
#!/bin/bash
#'consuma' la richiesta del client
while read line;
do
  if [ $(echo $line | wc -c) -lt "3" ]; then
    break
  fi
done
#scrive header del protocollo
echo HTTP/1.0 200 OK
echo "Content-Type: text/html; charset=utf-8"
echo "Connection: close"
DATE=$(date)
#prepara la risposta
response=$(sed -e "s/%DATE%/$DATE/g" /home/andrea/webserver/index.html)
length=$(echo $response | wc -c)
echo "Content-Length: $length"
echo
echo $response

modo interattivo

eseguire python

eseguendo l'interprete del linguaggio senza parametri otteniamo una shell che ci permette di dare comandi e vedere il risultato immediatamente. Comodo per fare delle prove veloci, ma non per scrivere programmi lunghi!

$ python
Python 3.4.2 (default, Oct  8 2014, 13:14:40)
[GCC 4.9.1] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>> print("ciao")
ciao

per uscire: premere CTRL-D oppure scrivere quit()

una shell piu' comoda iPython, che va installata a parte (vedremo come)
