shell script

gli argomenti passati allo script sono disponibili come $1 , $2, $3 eccetera oppure $* per tutti

$0 e' il nome dello script stesso

#!/bin/sh
echo "ciao, sono lo script $0 . Mi hai chiamato con parametri $1 e $2"

$# e' il NUMERO di argomenti

#!/bin/sh
echo "hai scritto $# cose: $*"

sostituzione di variabili

$VAR prende il contenuto della variabile e lo sostituisce nello script come se l'avessi scritto

$(comando) prende l'OUTPUT del comando e lo sostituisce nello script

espansione delle graffe (brace expansion) diretta e indiretta

$ export nome=andrea
$ export andrea=42

$ echo ${nome}  # normale
andrea

$ echo ${!nome} # indirezione: e' come se "nome" fosse un puntatore
42

espansione aritmetica: doppia tonda

z=$((z+3))

trucchi per scrivere buoni script

  • per capire come funziona uno script, fare una copia e modificare quella inserendo istruzioni di verifica
  • usare set -x per visualizzare l'esecuzione
  • non aver paura di commentare (in modo utile)
  • testare sempre le condizioni di errore e usare gli exit code in modo appropriato
  • inserire nello script una funzionalita' di help
  • cercare di fare funzioni "piccole" e riusabili
  • gli script devono essere idempotenti, cioe' rilanciandoli piu' volte ottengo lo stesso risultato
  • valutare l'uso di colori solo per script interattivi

http://www.davidpashley.com/articles/writing-robust-shell-scripts/

soluzione es2

#!/bin/sh
if [ -z "$1" ]; then
  echo "questo script serve per cambiare estensione a un gruppo di file."
  echo "uso: $0 vecchia nuova"
  exit
fi
for file in *.$1; do
   mv "$file" "$(basename $file .$1).$2"
done

per renderlo ricorsivo usare find, oppure con bash >= 4 si puo' usare il doppio ** per specificare globbing ricorsivo

proviamo un semplice servizio di rete

xinetd e' un servizio che fornisce internamente semplici demoni di rete.

apt-get install xinetd
vi /etc/xinetd.d/daytime  # metto disable a 'no'
systemctl start xinetd    # avvio il servizio
cat /etc/services|grep daytime
telnet 127.0.0.1 daytime

xinetd permette anche l'utilizzo via rete dei programmi concepiti per un I/O locale.
