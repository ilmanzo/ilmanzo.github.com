funzioni anonime (lambda)

in Python le funzioni sono un tipo di dato come gli altri: interi,stringhe,liste. Cio' significa che possiamo assegnare una variabile ad una funzione

def doppio(x):
  return x*2

g=doppio

#equivale a scrivere

g=lambda x: x*2

a cosa serve ?

l'utilizzo o meno delle funzioni lambda e' questione di stile. Usarle non e' mai necessario; in ogni caso in cui vengono usate e' possibile definire una funzione normale e usarla al posto della funzione lambda. Le funzioni lambda sono comode, ad esempio, nei casi in cui si voglia incapsulare codice specifico, non riutilizzato, senza riempire il programma di piccolissime funzioni.

Esempio:

  • passare una funzione ad un'altra funzione (callback)
  • manipolare liste

DECORATORI

argomento un po' complesso, procediamo passo per passo:

  1. in Python posso definire funzioni annidate, cioe' funzioni dentro funzioni
  2. una funzione Python accetta argomenti di vario tipo e puo' tornare un valore
  3. tra i tanti tipi di argomenti, posso passare anche funzioni e ritornare funzioni

un decoratore Python e' una sintassi alternativa per definire una funzione che:

  • accetta una funzione f() in ingresso
  • ritorna una nuova funzione che:
    • compie delle operazioni prima
    • esegue la funzione f() passata
    • compie delle operazioni dopo

https://realpython.com/blog/python/primer-on-python-decorators/

a cosa servono ?

"middleware": timing, autenticazione, controllo accessi ...

esempio di decoratori

import time

def time_it(some_function):
#decoratore che stampa il tempo impiegato a eseguire una certa funzione
  def wrapper():
    t1 = time.time()
    some_function()
    t2 = time.time()
    return "Time it took to run the function: " + str((t2 - t1)) + "\n"
  return wrapper

#utilizzo
@time_it
def my_function():
  num_list = []
  for num in (range(0, 10000)):
    num_list.append(num)
  print("\nSum of all the numbers: " + str((sum(num_list))))

print(my_function())

dai generatori alle coroutine

un semplice multitasking preemptive: scheduler che esegue a turno una serie di funzioni definite dall'utente ogni funzione mantiene i propri dati e il proprio stato (modello ad "attori")

def taskA(n,marker):
  for i in range(1,n):
    print("eseguo task A:",marker,i)
    yield

def taskB(n,marker):
  for i in range(n,0,-1):
    print("eseguo task B:",marker,i)
    yield

def taskC(n,marker):
  for i in range(1,n*2,2):
    print("eseguo task C:",marker,i)
    yield

tasks=[taskA(5,"pippo"),taskB(7,"pluto")]
tasks.append(taskC(3,"paperino"))
tasks.append(taskB(4,"paperoga"))

while True: # "motore" di esecuzione    
  for t in tasks:
    try:
      next(t)
    except StopIteration:
      pass
