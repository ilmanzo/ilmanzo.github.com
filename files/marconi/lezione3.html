il "nodo"

un flow Node-RED funziona facendo passare messaggi tra i nodi. Un "messaggio" non e' altro che un oggetto javascript che ha una serie di proprieta', ad esempio "payload"

in Javascript le variabili possono avere valori semplici, come 

var x = 5;
var y = 6;
var z = x + y;
var nomeAuto = "Volvo";

oppure contenere una serie di propriet

var persona = { 
nome: "Mario",
cognome: "Rossi",
indirizzo: {
    via: "Via Garibaldi",
    numero: 15,
    CAP: "00100",
    citta: "Roma"
    }
};

var nome = persona.nome;
var nome = persona["nome"];
persona.eta = 32;
function visualizzaNomeCognome() { return "Mario Rossi"; }
persona.nomeCognome = visualizzaNomeCognome; // o perfino funzioni!

panoramica sui nodi

nodi di controllo logico di flusso

  • nodo test: esegue un confronto, pu avere tanti output che variano a seconda delle condizioni
  • nodo change: modifica il messaggio; set/delete propriet
  • nodo range: accetta come payload solo numeri e mappa intervalli di input in intervalli di output
  • nodo split: divide il payload di un messaggio (ad esempio una stringa o un array) in tanti messaggi
  • nodo join: riunisce N messaggi che abbiano la propriet parts
  • nodo sort: puo' mettere in ordine una property di un messaggio singolo, o un flusso di messaggi che abbiano la propriet parts (vedi split/join)
  • nodo batch: pu raggruppare sequenze di messaggi secondo diversi criteri

node-red all'avvio

per far partire automaticamente Node-RED all'avvio del sistema, dovremo usare un systemd service, o nei sistemi pi vecchi, uno script di init. 

[Unit]
Description=Node-RED
After=syslog.target network.target

[Service]
ExecStart=/usr/bin/node-red --max-old-space-size=128 -v
Restart=on-failure
KillSignal=SIGINT

# log output to syslog as 'node-red'
SyslogIdentifier=node-red
StandardOutput=syslog

# non-root user to run as
WorkingDirectory=/home/andrea/
User=andrea
Group=andrea

[Install]
WantedBy=multi-user.target
