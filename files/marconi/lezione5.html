soluzione

inserisco un nodo "error" che catturera' l'errore e mi permette di propagarlo ad altri nodi (inviarlo via mail, scrivere in un file di log, aggiornare una pagina web, eccetera)

var temp=parseFloat(msg.payload)
if (temp<-30 || temp>50 ) {
    node.error("anomalia temperatura:"+temp, msg);
    msg.payload="ERR";
}
return msg;

in modo simile posso inserire anche un nodo "status"

esercizio

inseriamo in un nuovo flow un nodo 'timestamp' che emetta un messaggio ogni secondo.

Colleghiamo al timestamp una funzione che cambi lo stato del nodo a seconda se il timestamp pari o dispari. 

if ((msg.payload & 1)===0) {
    node.status({fill:"red", text:"pari"});
    msg.payload=0
} else {
   node.status({fill:"green", text:"dispari"});
   msg.payload=1
}
return msg;

messaggi e contesto

oltre a usare le propriet payload, una funzione pu memorizzare dati in un contesto. Ad esempio ecco un nodo funzione che "conta" quante volte viene invocato e visualizza il conteggio nel proprio status:

// inizializza il contatore a zero
var count = context.get('count')||0;
count += 1;
// memorizza il valore nel contesto
context.set('count',count);
// visualizza nello status: icona verde se pari, rossa se dispari
var colore=(count % 2==1)?"red":"green";
node.status({fill:colore,shape:"dot",text:count});
return msg;

il contesto e' "locale" al nodo (ovvero ogni nodo ha il suo); esiste anche un contesto a livello di flow (si setta con flow.set/get) e uno 'globale' condiviso tra tutti i flow: global.set/get.

Il contesto normalmente viene 'dimenticato' ad ogni restart di Node-RED, ma possibile renderlo persistente.

invocare codice esterno

grazie al nodo exec , posso eseguire qualsiasi programma sul computer e avere in msg.payload l'output ; inoltre viene salvato il codice di uscita del programma.

questo ci permette di espandere il nostro flow prelevando e immettendo dati anche all'esterno dell'ambiente Node-RED

notare che il nodo EXEC ha 3 uscite: standard output, standard error, exit code del comando

#legge la temperatura della CPU del sistema, in BASH
TEMPCPU=$(cat /sys/class/thermal/thermal_zone0/temp)
echo $(( $TEMPCPU/1000 ))

#stessa cosa in python3
with open('/sys/class/thermal/thermal_zone0/temp', 'r') as infile:
  temp=infile.readline()
  print(float(temp)/1000.0)

catturare l'output

quando un processo invoca un altro processo ( syscall: fork() + exec() ) pu modificare i file descriptor per manipolare l'input o intercettarne l'output

stdin, stdout, stderr

redirezione dell'input O output

se non diversamente specificato, l'output di un comando shell viene emesso a video e l'input arriva dalla tastiera (terminale)

per mandare l'output su un file:

comando > file.txt

per riceve l'input da un file:

comando < file.txt

redirezione dell'input E output

stdout e stderr

i programmi hanno un canale di output "normale" (/dev/stdout) e uno per gli errori (/dev/stderr)

per default entrambi finiscono a video, possiamo redirigere uno dei due su un file

grep da * 2> grep-errors.txt

oppure mandare lo stdout su stderr

grep da * 1>&2

o viceversa

grep * 2>&1

o entrambi nello stesso file:

find / -name core &> /dev/null
